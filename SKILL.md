# 论文格式转换器

将 Markdown 论文稿按指定学术格式规范一键转换为排版规范的 DOCX 文档。

## 核心价值

学术格式规范的细节极其繁琐——字体字号行距边距、标题层级、引用格式、章节划分，每个节点都有精确要求。这个技能的核心价值不在于"md→docx 转换"（那是工具的事），而在于**将论文格式权威标准翻译为可执行的排版参数**，让标准本身驱动输出。

## 工作流程

### 步骤一：接收输入

用户提供：
- **Markdown 论文文件**（包含完整论文内容，含标题、摘要、正文、参考文献等）
- **目标格式规范**（如 "GB/T 7714-2025 学位论文"、"APA 7th"、"MLA 9th" 等）
- **可选：自定义覆写参数**（如特定高校的排版要求）

### 步骤二：确定格式参数

根据目标格式规范，从内置预设中加载对应的排版参数映射。参数覆盖四个维度：

| 维度 | 涵盖内容 | 依据标准 |
|------|---------|---------|
| 宏观结构 | 封面/摘要/目录/正文/参考文献等章节划分与顺序 | GB/T 7713.1-2006、IMRaD |
| 排版样式 | 字体族、字号、行距、页边距、对齐方式 | 高校学位论文规范、APA/MLA 手册 |
| 引用格式 | 文中标注方式、参考文献列表格式 | GB/T 7714-2025、APA 7th、MLA 9th、Chicago |
| 图表公式 | 图注/表题位置、公式编号、编号层级 | GB/T 7713.1、各高校规范细则 |

#### 内置预设格式

| 格式键 | 名称 | 适用场景 |
|--------|------|---------|
| `gbt7714` | GB/T 7714 学术论文格式（中文） | 中文学术论文通用排版规范 — 宋体小四正文、黑体标题、首行缩进2字符、1.5倍行距 |
| `apa7` | APA 7th Edition | 社会科学期刊论文 |
| `mla9` | MLA 9th Edition | 人文学科论文 |
| `chicago` | Chicago Notes-Bibliography | 历史学等脚注密集型学科 |
| `imrad` | IMRaD 国际结构 | 自然科学/医学论文 |

### 步骤三：执行转换

调用 `main.py` 脚本，传入 Markdown 文件和格式参数，脚本自动：
1. 加载格式预设 → 生成本次排版参数（写入 `style` 子对象 + `template.page.margin`）
2. 生成 md-to-docx 的 options.json 配置文件
3. 调用 `npx @mohtasham/md-to-docx` 执行转换
4. **gbt7714 格式额外执行中文后处理：**
   - 正文段落：宋体小四(12pt)，1.5倍行距，首行缩进2字符，两端对齐
   - 一级标题：黑体三号(16pt)加粗，居中，段前24pt段后12pt
   - 二级标题：黑体四号(14pt)加粗，左对齐，段前12pt段后6pt
   - 三级标题：黑体小四(12pt)加粗，左对齐，段前6pt段后3pt
   - 英数自动使用 Times New Roman
5. 输出符合规范的 DOCX 文件

### 步骤四：交付与验证

- 向用户输出 DOCX 文件路径
- 告知应用的格式规范和关键排版参数摘要
- 用户检查后如需调整，可通过 `--custom-options` 覆写特定参数

## 命令格式

```bash
python main.py --input paper.md --output paper.docx --format gbt7714

# 带自定义覆写
python main.py -i paper.md -o paper.docx -f gbt7714 -c my_options.json

# 列出所有可用格式
python main.py --list-formats
```

## 自定义 options 说明

内置预设覆盖大部分通用场景。如果用户有特殊排版要求（如某高校要求"正文宋体小四、行距固定值22磅、一级标题黑体三号"），可以通过 `--custom-options` 传入 JSON 文件覆写特定参数。

关键参数说明：
- `fontFamily`：正文字体（如 "宋体"、"Times New Roman"）
- `headingFontFamily`：标题字体（中文学位论文通常为黑体）
- `paragraphSize`：正文字号（pt）
- `lineSpacing`：行距倍数
- `pageMarginTop/Bottom/Left/Right`：页边距（cm）
- `headingAlignment`：标题对齐方式

## 边界情况处理

- **输入文件不存在**：报错并提示正确路径
- **未知格式名**：列出所有可用格式供用户选择
- **md-to-docx 未安装**：首次运行 npx 会自动安装，如网络问题导致失败，提示用户手动 npm install
- **参考文献格式**：当前版本预设文档级排版参数，参考文献条目格式（如顺序编码制 vs 著者-出版年制）需在 Markdown 中按规范提前编排
- **数学公式**：md-to-docx 原生支持 `$...$` 和 `$$...$$`，直接在 Markdown 中使用即可
- **自定义高校规范**：如果用户的高校有独特排版要求，通过 `--custom-options` 传入 JSON 覆写参数，或在 Markdown 中通过 HTML 注释标注特殊需求

## 注意事项

- 输入的 Markdown 应包含完整的论文结构（标题层级用 `#` 表示，与 DOCX 标题样式对应）
- 参考文献建议在 Markdown 中用有序列表编排，引用标记与正文对应
- 图表用标准 Markdown 语法 `![图注](path)`，图注格式由 md-to-docx 控制
- 如果需要分节文档（如前言的页码格式与正文不同），使用 md-to-docx 的 sections 功能在自定义 options 中配置

## 中文论文格式标准（gbt7714）

依据 **GB/T 7713.1-2025** + 高校通用规范：

| 元素 | 字体 | 字号 | 对齐 | 段前 | 段后 | 特殊 |
|------|------|------|------|------|------|------|
| 一级标题（章） | 黑体 | 三号 16pt | 居中 | 24pt | 12pt | **每一章另起页** |
| 二级标题（节） | 黑体 | 四号 14pt | 左对齐 | 12pt | 6pt | — |
| 三级标题（子节） | 黑体 | 小四 12pt | 左对齐 | 6pt | 3pt | 加粗 |
| 正文 | 宋体 | 小四 12pt | 两端对齐 | 0 | 0 | 首行缩进2字符，1.5倍行距 |
| 英/数字 | Times New Roman | 与上下文一致 | — | — | — | — |

**页边距**：上2.5cm、下2.5cm、左3.0cm（装订侧）、右2.0cm（A4纸）。

**关键规范来源**：
- §5.3.1：正文部分应从另页右页开始，每一章应另起页
- §6.10：天头≥25mm，订口≥25mm，地角≥20mm，切口≥20mm
- §6.5：章的标题占2行，正文另起行前空2字

### 实现说明

中文格式通过 **md-to-docx + python-docx 双引擎** 实现：
1. md-to-docx 做结构转换（标题层级、段落、表格、代码块）
2. python-docx 后处理校正所有中文特有格式：
   - 设置中文字体（eastAsia 属性，宋体/黑体）
   - 设置西文字体（Times New Roman）
   - 首行缩进2字符（480t
